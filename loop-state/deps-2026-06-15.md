# Dependency Security Upgrade — 2026-06-15

## Scope
Patch only **security-critical** vulnerabilities (severity **high** or **critical**) reported in `loop-state/audit.json`. No other version changes.

## Vulnerabilities found

| Package | Severity | Direct? | In scope? |
|---|---|---|---|
| `next` | **high** | yes | ✅ patched |
| `postcss` | moderate | no (transitive of `next`) | ❌ below threshold — not touched |

No `critical` vulnerabilities were present.

## Change applied

| Package | From | To | Type |
|---|---|---|---|
| `next` | `14.2.29` | `15.5.16` | major bump (security) |

`next` was pinned to an exact version in `package.json`; the same exact-pin style is preserved.

### Why `15.5.16`
`15.5.16` is the **minimal** version that clears every high-severity advisory affecting `next`. The 14.2.x line cannot resolve all of them (e.g. RSC deserialization DoS requires ≥15.0.8). The high-severity advisories addressed:

- GHSA-mwv6-3258-q52c — DoS with Server Components (fixed 14.2.34)
- GHSA-5j59-xgg2-r9c4 — DoS with Server Components, follow-up (fixed 14.2.35)
- GHSA-h25m-26qc-wcjf — HTTP request deserialization DoS via insecure RSC (fixed 15.0.8)
- GHSA-q4gf-8mx6-v5v3 — DoS with Server Components (fixed 15.5.15)
- GHSA-8h8q-6873-q5fj — DoS with Server Components (fixed 15.5.16)
- GHSA-c4j6-fc7j-m34r — SSRF via WebSocket upgrades, CVSS 8.6 (fixed 15.5.16)
- GHSA-36qx-fr4f-26g5 — Middleware/Proxy bypass in Pages Router i18n (fixed 15.5.16)

Bumping to `15.5.16` also incidentally resolves the bundled `postcss` advisory and the moderate `next` advisories, since they are all fixed at or below this version.

## Out of scope (intentionally unchanged)
- `postcss` — moderate severity, transitive, not in `package.json`. Updated automatically as a `next` sub-dependency.
- `@anthropic-ai/sdk`, `framer-motion`, `react`, `react-dom`, `@types/*`, `typescript` — outdated but not flagged as high/critical vulnerabilities; left untouched.

## Follow-up notes
- This is a **major version bump** (Next.js 14 → 15). It requires `npm install`, a build, and test/manual verification before merge — App Router behavior, middleware, and image optimization changed across the major. npm's `audit fix` recommends `next@16.2.9`; `15.5.16` was chosen as the smaller, security-sufficient step.
